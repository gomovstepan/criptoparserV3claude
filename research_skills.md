# Исследование скиллов для крипто-арбитражной системы

## Резюме
- **Всего скиллов проанализировано:** 261
- **Критичных:** 42
- **Полезных:** 77
- **Не применимых:** 142

Проект: закрытая SaaS-система крипто-арбитражной торговли (Python 3.11+ / FastAPI / React 19 / TimescaleDB / Redis Streams / Docker Compose / 7 криптобирж / Telegram бот / React-дашборд).

## Недостающие скиллы

Ниже перечислены скиллы, которые **необходимы для проекта**, но **отсутствуют** в системе:

### 1. **WebSocket Client Skill** 🔴
- **Почему нужен:** Проект подключается к 7 криптобиржам через WebSocket для real-time данных. Нужен специализированный скилл для управления WebSocket-соединениями (reconnection, heartbeat, multiplexing).
- **Применение:** Модули приема котировок (market data adapters).

### 2. **Redis Streams Skill** 🔴
- **Почему нужен:** Redis Streams используется как message bus между микросервисами. Нужен скилл для работы с Redis Streams (publish/subscribe, consumer groups, stream processing).
- **Применение:** Межсервисная коммуникация, event sourcing.

### 3. **TimescaleDB/Hypertable Skill** 🔴
- **Почему нужен:** Хотя есть `database-inspector` и `sql-insight`, нет специализированного скилла для Time-series оптимизаций TimescaleDB (hypertables, continuous aggregates, retention policies, chunk management).
- **Применение:** Хранение tick-данных, агрегация OHLCV.

### 4. **Telegram Bot Skill** 🔴
- **Почему нужен:** Telegram бот используется для уведомлений о сделках и арбитражных возможностях. Нужен скилл для разработки Telegram ботов (aiogram/ python-telegram-bot).
- **Применение:** Модуль уведомлений.

### 5. **Docker Compose / Container Orchestration Skill** 🔴
- **Почему нужен:** Проект использует Docker Compose для деплоя. Нужен скилл для мультисервисной оркестрации, networking, health checks, volume management.
- **Применение:** DevOps, деплой, локальная разработка.

### 6. **Crypto Exchange API Adapter Skill** 🔴
- **Почему нужен:** Нужен шаблонизированный скилл для создания адаптеров к криптобиржам (Binance, Coinbase, Kraken, etc.) — унифицированный интерфейс для REST + WebSocket API, аутентификация, rate limiting.
- **Применение:** Модуль интеграции с биржами.

### 7. **Real-time Data Processing / Stream Processing Skill** 🔴
- **Почему нужен:** Обработка потоковых данных котировок в real-time (windowing, stateful processing). Нет специализированного скилла для stream processing.
- **Применение:** Детекция арбитражных возможностей в реальном времени.

### 8. **Financial Risk Management Skill** 🟡
- **Почему нужен:** Расчет VaR, максимального drawdown, position sizing, stop-loss стратегии. `risk-heatmap` есть, но нет глубокого скилла риск-менеджмента.
- **Применение:** Управление рисками торговых стратегий.

### 9. **React 19 + TypeScript Specialized Skill** 🟡
- **Почему нужен:** `webapp-building` покрывает общий случай, но нет специализированного скилла для React 19 (Server Components, Suspense, новые хуки) + финансовых дашбордов (реал-тайм обновления, WebSocket в React).
- **Применение:** Frontend дашборд.

### 10. **Arbitrage Detection Algorithm Skill** 🔴
- **Почему нужен:** Ядро системы — алгоритмы поиска арбитражных возможностей (triangular arbitrage, cross-exchange arbitrage). Нет специализированного скилла для арбитражной математики.
- **Применение:** Core trading engine.

### 11. **API Key / Secrets Management Skill** 🟡
- **Почему нужен:** Управление API ключами бирж (ротация, шифрование, vault). `code-safety-audit` находит утечки, но нет скилла для правильного управления секретами.

### 12. **Order Execution / Trade Lifecycle Skill** 🔴

- **Применение:** Execution module.

## Примечания

### Особенности использования скиллов в контексте проекта

1. **Дублирование скиллов:** Некоторые скиллы имеют дубли (EN/CN версии): `code-safety-audit`/`code-vuln-audit`, `api-doc-gen`/`route-to-openapi`, `iteration-planner`/`sprint-plan-builder`. В проекте достаточно использовать одну версию в зависимости от языка команды.

2. **Трейдинговая специфика:** Скилл `trading-strategy-backtest` — единственный имеющий прямое отношение к трейдингу. Он критичен, но покрывает только бэктестинг. Для production торговли нужны дополнительные скиллы (см. Недостающие).

3. **Безопасность:** 5 критичных скиллов безопасности (`code-safety-audit`, `code-vuln-audit`, `secure-code-review`, `web-security-audit`, `compliance-review-planner`) должны запускаться на каждый релиз.

4. **Визуализация:** 4 критичных скилла визуализации (`chart-gen`, `chart-image`, `data-viz-gen`, `data-viz-renderer`) покрывают разные сценарии: PNG-графики для уведомлений и HTML/SVG для дашборда.

5. **Микросервисы:** Docker Compose выбран вместо Kubernetes, поэтому `k8s-cluster-ops` и `kubectl` не применимы. Нужен скилл Docker Compose (см. Недостающие).

6. **Мультиязычность:** Проект предполагает интерфейс на нескольких языках, но `locale-guard` и `localization-toolkit` имеют низкий приоритет для MVP.
